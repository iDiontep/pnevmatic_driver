# pnevmatic_driver — прошивка STM32 для оси на TB6560

В каталоге **`Magic`** лежит проект **STM32CubeIDE** для STM32F373: STEP/DIR/EN на драйвер **BL‑TB6560‑V2.0**, обмен с ПК по **USB CDC** (виртуальный COM), два концевика, сохранение настроек во Flash и абсолютное позиционирование в логических шагах.

Подробнее про распайку CLK/CW/EN и полярность **EN**: см. [**Magic/README.md**](Magic/README.md).

---

## Железо (кратко)

| Функция        | МК (этот проект) |
|----------------|------------------|
| STEP (PWM)     | **PA0** — TIM2 CH1 |
| DIR            | **PA1**          |
| EN             | **PA2**          |
| Концевик «мин» | **PA3**, активный LOW, подтяжка вверх |
| Концевик «макс»| **PA4**, активный LOW, подтяжка вверх |

Смысл «логического минимума/максимума» оси задаётся **`APS POSITION_DIR`** (`1` или `-1`): при `-1` роли концевиков для логики мин/макс меняются местами (см. `limits.h`).

Полярность линии **EN** при необходимости правится макросом **`TB6560_EN_ASSERTED_LOW`** в `Magic/Hardware_drivers/Inc/tb6560.h`.

## Сборка и прошивка

1. Установите **STM32CubeIDE**.
2. **File → Open Projects from File System** и укажите каталог **`Magic`** (или импортируйте как существующий проект).
3. Сборка: **Project → Build Project** (конфигурация **Debug** или **Release**).
4. Прошивка и отладка: **Run / Debug** через ST‑Link (как настроено в `.launch`/проекте).

Точка входа: `Magic/Core/Src/main.c` — инициализация периферии, загрузка APS из EEPROM, `tb6560_init`, при необходимости калибровка концевиков, затем цикл с `app_process()` и приёмом USB.

## Настройки и данные

### APS (`app.settings`)

Хранятся в **`app_settings_t`** (`Magic/Logic/Inc/app.h`): статус калибровки, `position_min` / `position_max`, направление оси `position_dir`, скорость по умолчанию `motor_speed` (Гц STEP), параметры рампы MOVE `ramp_step_interval`, `ramp_hz_step`, `ramp_min_hz`.

- **`APS_STATUS_DFLT`** (`0x000B00B5`) — дефолт / ещё не откалибровано.
- **`APS_STATUS_CALIB_OK`** (`0x000FACE`) — после успешной калибровки по концевикам.

Загрузка при старте: **`eeprom_try_load()`**; запись в последнюю страницу внутренней Flash — **`eeprom_save()`** (см. `Magic/Hardware_drivers/Inc/eeprom.h`).

### APD (`app.data`)

Рабочие данные (`minutes`, `button`, `current_position`) — по протоколу ниже.

## USB CDC — протокол команд

Строки **ASCII**, завершение **CR** и/или **LF**. Строка должна прийти целиком (при фрагментированной передаче без перевода строки разбор не выполняется).

Команды начинаются с префикса **`EFGH`** (регистр префикса не важен); хвост строки перед разбором приводится к верхнему регистру. Токены разделяются **пробелом**, **`,`** или **`=`**.

| Исход | Успех | Ошибка |
|-------|--------|--------|
| распознанный `EFGH …` | ответ вида `APS …`, `MOT …`, `POSITION …` и т.д. + CRLF | строка **`HGFE Error - …`** + перевод строки |
| строка без префикса `EFGH` после trim | эхо входных данных обратно на CDC | — |

Детальный контракт (все подкоманды APS / APD / MOT / POSITION): комментарии в **`Magic/Hardware_drivers/Inc/receiver.h`**.

Примеры:

```text
EFGH GET APS ALL
EFGH SET MOT MOVE 1000 800
EFGH SET MOT STOP
EFGH POSITION 500
EFGH GET MOT STAT
```

Команды **`POSITION`**, **`MOT MOVE`**, **`MOT HZ`**, **`MOT STOP`** отменяют текущее движение; отдельного флага «busy» в протоколе нет.

## Движение и концевики

- **`MOT MOVE`** — заданное число импульсов STEP с разгоном и торможением; параметры рампы задаются APS и **`tb6560_set_move_ramp()`**.
- **`MOT HZ`** — непрерывное вращение на заданной частоте (**RUN**).
- **`POSITION x`** — блокирующая поездка к абсолютной логической позиции в допустимом диапазоне **APS** (нужна валидная калибровка).

При наезде на концевик **в сторону упора**:

- в режиме **MOVE** прошивка стремится **притормозить по рампе** (укорачивается хвост команды до фазы торможения), затем останов и смена направления DIR для возможного отъезда от упора;
- в режиме **RUN** выполняется **немедленная остановка** и разворот DIR.

Калибровка границ оси по концевикам выполняется модулем **`service_calibrate_limits()`** (`Magic/Logic/Src/service.c`), вызывается из инициализации при выполнении условий в **`main.c`** (см. актуальный код).
